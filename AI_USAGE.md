# AI 활용 내역 보고서 (AI Usage Report)

본 프로젝트는 개발자가 주도적으로 개발을 진행하되, 개발 생산성 향상과 코드 퀄리티 확보를 위해 **Google Gemini**를 기술 파트너로 활용하였습니다.

단순 코딩보다는 **아키텍처 설계 검증, 복잡한 로직의 최적화, 그리고 디버깅** 영역에 집중하여 AI를 활용했습니다.

## 1. 활용 개요

| 구분              | 활용 도구  | 주요 활용 내용                                 | 비중 |
| :---------------- | :--------- | :--------------------------------------------- | :--- |
| **아키텍처/구조** | **Gemini** | MPA 폴더 구조 검토                             | 30%  |
| **핵심 로직**     | **Gemini** | 데이터 무결성 로직(Storage)                    | 30%  |
| **디버깅/최적화** | **Gemini** | Git 병합 충돌 해결, Tailwind v4 설정 이슈 해결 | 30%  |
| **리소스 제작**   | **Gemini** | 앱 로고, 파비콘(SVG), 캐릭터 썸네일 생성       | 10%  |

## 2. 주요 활용 사례

### A. 데이터 관리 로직 고도화

- **활용 배경:** DB 없이 LocalStorage만으로 관계형 데이터(User-Chat-Character)를 안정적으로 관리해야 하는 기술적 요구사항 존재.
- **적용 내용:**
  - **가상 DB 계층 구현:** 단순 저장 로직을 넘어, 데이터 간의 연관 관계(Foreign Key 유사 동작)를 처리하는 `storage.ts` 서비스 레이어 구현 시 활용.

### B. 백엔드 API 통합 및 프록시 서버 구현

- **활용 배경:** 클라이언트(Frontend)에서 직접 Claude API를 호출할 경우 API Key가 노출되는 보안 위험이 있어, 별도의 중계 서버(Proxy Server) 구축이 필요.
- **적용 내용:**
  - **Secure Proxy Pattern:** Express 프레임워크를 사용하여 프론트엔드의 요청을 받아 API Key를 헤더에 태워 Claude 서버로 전달하는 프록시 라우트를 Gemini와 함께 구축.
  - **Context Awareness Logic:** 단순히 마지막 메시지만 보내는 것이 아니라, `storage`에 저장된 이전 대화 내역(`history`)을 `messages` 배열 형태로 구조화하여 API에 전달함으로써, AI가 이전 대화의 맥락을 기억하도록 하는 로직 구현.

### C. 디자인 자산 생성 (Design Assets)

- **활용 배경:** SVG 그래픽 및 시각 자료 생성.
- **적용 내용:**
  - **Gemini (NanoBanana):** 프로젝트의 아이덴티티를 살린 앱 로고 및 파비콘 SVG 코드를 생성하여 최적화 적용.
  - **UI 리소스:** 기본 제공 캐릭터의 페르소나에 맞는 썸네일 이미지 생성.

## 3. 결론

AI를 단순한 코드 생성기가 아닌, 설계 검증 및 디버깅 시 활용했습니다. 이를 통해 비즈니스 로직과 사용자 경험(UX) 구현에 더 집중할 수 있었으며, 결과적으로 제한된 시간 내에 안정적이고 완성도 높은 애플리케이션을 구축할 수 있었습니다.
